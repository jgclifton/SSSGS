SSSGS is (yet another) Sass grid system.

It's designed for each user to change as they need, and contains no other styling such as typography...

... and yes, it's responsive :)

Just a Grid

Super simple. Under 150 lines of code - it's easy to read and modify and contains no other styling. All nestable by default and no need to use alpha/omega or first/last classes.

Easy to Configure

SSGS is designed to be modified by each user - download it and make it your own. Specify the amount of columns and how big you want gutters to be.

Easy Syntax

There's no fractions here, just a simple number system - .column-4-16 means "take up four columns out of the sixteen available".

Why another grid system?

Making a grid system is like a right of passage for a web developer. Just kidding. This grid system is really self serving, it has been released just to give something back, and see if others like it.

It's really small and easy to configure, with just a few minutes of practice you can create flexible grids in no time.

It's currently still in BETA, so do let us know if you get any issues.

Key Features

  • Uses Sass (scss variety)
  • It's just a grid: solid, lightweight and not intrusive.
  • Specify the amount of columns and gutter widths that you need.
  • It's desktop first out of the box, but easy to change if you want mobile first.
  • Contains just a few useful helper classes.
  • Specify how many columns and gutter widths in seconds.
    • Uses box-sizing: border-box; so ie8+ only.

Installation and Usage

1 Download the .scss file and drop it in your project. It's recommended to have this is a sperate file and then import it using the sass import function.

import "grid";

2 Open up the grid.scss file and start to change some settings. By default it has a 1200px container, 16 columns and 10px gutters. Change the $columns and $gutters variables to what you need, and compile the sass.

3 We have three simple concepts to use the grid: containers, rows and columns. The .container is what wraps around all of your content (and by default on a large screen is 1200px wide).

Rows contain columns and you need a row each time you've used up all of your columns.

Take a look at the adjoining code sample to see how this works. Unsurprisingly, this site is using SSSGS, so take a look at the source code for more samples.

There's also some generic sizes include for those with column numbers that don't divide in to third well. These are: .column-one-quarter, .column-one-third .column-half, .column-three-quarters, .column-two-thirds.

4 Offsets? no problem! Offsets follow a very similar syntax to columns. Take a look at the adjoining code sample to see them in action.

4 Now lets look at some breakpoints, because everyone loves responsive, right? SSSGS ships with just a few generic breakpoints, but adding new ones is straight forward.

There's a few simple steps that can be followed. First, set up your new media query and then set your new container width. Take a look at the adjoining code snippet to see what defaults are included.

SSSGS drops down to mobile (single column) at 720px - if this is too large or too small, simply edit the breakpoint in grid.scss.

5 Get some help. There's some helper classes included that may be handy in a few situations.

  • Add a .stick class to make columns keep their proportions - this will need to be applied to all columns in the row
  • Remove gutters with .no-gutters, .no-gutter-left and no-gutter-right.
